运维工程师面试指南

这份资料涵盖了运维工程师面试中常见的知识点和问题，包括Linux基础、Shell脚本、容器技术、监控系统等多个方面， 帮助你全面准备面试，顺利拿到心仪的offer。

建议学习时长: 2-3周 知识点: 100+ 面试题: 200+

Linux基础

核心知识点

  • 文件系统: ext4、xfs特性及区别，inode结构，文件权限管理
  • 进程管理: 进程状态（R、S、D、Z、T），进程调度，fork/exec机制
  • 内存管理: 虚拟内存原理，页表机制，OOM killer，swap空间
  • 用户与权限: UID/GID，文件权限（rwx），ACL，sudo配置
  • 服务管理: systemd vs sysvinit，服务启动级别，unit文件
  • 包管理: rpm/yum，dpkg/apt，源码编译安装

常用命令

文件操作

ls, cd, pwd, mkdir, rm, cp, mv, touch, ln

文本处理

cat, grep, sed, awk, head, tail, less, wc

进程管理

ps, top, htop, pstree, kill, pkill, pgrep

系统信息

df, du, free, vmstat, iostat, netstat, ss

面试高频问题

1. 如何查看系统中占用CPU最高的进程？

使用top命令（按P排序）或ps aux --sort=-%cpu | head -n 10

2. 什么是inode？它包含哪些信息？

inode是文件系统中的数据结构，包含文件元数据（权限、所有者、大小、时间戳等），但不包含文件名。

3. 如何查找近7天内被修改过的文件？

find /path -mtime -7

4. 解释一下Linux系统的启动流程？

BIOS/UEFI → 引导加载程序(GRUB) → 内核初始化 → systemd初始化 → 登录界面

5. 如何限制用户的磁盘空间使用？

使用quota工具，配置用户或组的磁盘配额

面试高频问题

1. $*和$@的区别是什么？

在不加引号时没有区别，都表示所有参数；加引号时，"$*"将所有参数视为一个整体，"$@"将每个参数视为独立个体。

2. 如何在脚本中捕获信号（如Ctrl+C）？

使用trap命令，例如：trap 'echo "Ctrl+C pressed"' SIGINT

3. 如何判断一个文件是否存在？

使用if [ -f "filename" ]; then ... fi

4. 如何在脚本中处理命令行参数？

使用位置参数$1, $2等，或使用getopts处理选项参数

5. 写一个脚本，监控服务器磁盘使用率，超过80%则报警

df -h | awk 'NR>1 {gsub("%",""); if($5>80) print "Disk " $1 " usage is " $5 "%"}'

面试高频问题

1. 容器与虚拟机的区别是什么？

容器共享宿主机内核，启动快、资源占用少；虚拟机有独立内核，隔离性更好但资源开销大。

2. Docker镜像的分层结构是什么？有什么好处？

镜像采用分层文件系统，每层只读，容器启动时添加可写层。好处：共享相同层，节省空间；便于版本控制和复用。

3. 什么是Kubernetes的Pod？为什么需要Pod？

Pod是K8s最小部署单元，包含一个或多个容器，共享网络和存储。用于部署紧密关联的容器，确保它们在同一节点运行并协同工作。

4. Deployment和StatefulSet的区别？

Deployment用于无状态应用，Pod名称随机；StatefulSet用于有状态应用，提供稳定网络标识和有序部署/扩展/更新。

5. 如何实现Kubernetes集群的高可用？

多master节点，使用负载均衡；etcd集群（奇数节点）；关键组件多副本部署；使用PVC保证数据持久化。

面试高频问题

1. Prometheus的数据模型是什么样的？

时间序列数据，由指标名称(metric name)和标签(labels)唯一标识，每个时间序列是一组带有时间戳的值。

2. 如何设计一个完整的监控体系？

基础设施监控→应用监控→业务监控→用户体验监控，结合日志、指标、追踪数据，建立完善的告警机制和可视化。

3. 什么是PromQL？举例说明其用途。

Prometheus查询语言，用于从时间序列数据中查询和聚合数据。例如：http_requests_total{status="200"} 表示所有状态码为200的HTTP请求总数。

4. 如何处理监控系统的告警风暴？

设置告警抑制和分组，按业务优先级排序，实现告警聚合，避免重复告警，建立告警升级机制。

5. ELK Stack中各组件的作用是什么？

Elasticsearch：存储和索引日志；Logstash：收集和处理日志；Kibana：日志可视化和查询分析。

面试高频问题

1. MySQL主从复制的原理是什么？

主库开启binlog，从库通过IO线程读取主库binlog写入relay log，SQL线程执行relay log中的事件，实现数据同步。

2. 如何优化MySQL查询性能？

添加合适的索引，避免全表扫描；优化SQL语句，避免使用select *；分库分表；调整MySQL参数；增加缓存等。

3. Redis的持久化机制有哪些？区别是什么？

RDB：定时快照，性能好但可能丢失数据；AOF：记录所有写操作，数据安全性高但文件大，恢复慢。

4. 什么是数据库事务的ACID特性？

原子性(Atomicity)、一致性(Consistency)、隔离性(Isolation)、持久性(Durability)。

5. 如何处理MySQL主从延迟？

优化主库写入性能；升级从库硬件；使用并行复制；避免大事务；考虑使用中间件如MyCat、ProxySQL等。
