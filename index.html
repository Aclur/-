面试高频问题

1. 如何查看系统中占用CPU最高的进程？

使用top命令（按P排序）或ps aux --sort=-%cpu | head -n 10

2. 什么是inode？它包含哪些信息？

inode是文件系统中的数据结构，包含文件元数据（权限、所有者、大小、时间戳等），但不包含文件名。

3. 如何查找近7天内被修改过的文件？

find /path -mtime -7

4. 解释一下Linux系统的启动流程？

BIOS/UEFI → 引导加载程序(GRUB) → 内核初始化 → systemd初始化 → 登录界面

5. 如何限制用户的磁盘空间使用？

使用quota工具，配置用户或组的磁盘配额

面试高频问题

1. $*和$@的区别是什么？

在不加引号时没有区别，都表示所有参数；加引号时，"$*"将所有参数视为一个整体，"$@"将每个参数视为独立个体。

2. 如何在脚本中捕获信号（如Ctrl+C）？

使用trap命令，例如：trap 'echo "Ctrl+C pressed"' SIGINT

3. 如何判断一个文件是否存在？

使用if [ -f "filename" ]; then ... fi

4. 如何在脚本中处理命令行参数？

使用位置参数$1, $2等，或使用getopts处理选项参数

5. 写一个脚本，监控服务器磁盘使用率，超过80%则报警

df -h | awk 'NR>1 {gsub("%",""); if($5>80) print "Disk " $1 " usage is " $5 "%"}'

面试高频问题

1. 容器与虚拟机的区别是什么？

容器共享宿主机内核，启动快、资源占用少；虚拟机有独立内核，隔离性更好但资源开销大。

2. Docker镜像的分层结构是什么？有什么好处？

镜像采用分层文件系统，每层只读，容器启动时添加可写层。好处：共享相同层，节省空间；便于版本控制和复用。

3. 什么是Kubernetes的Pod？为什么需要Pod？

Pod是K8s最小部署单元，包含一个或多个容器，共享网络和存储。用于部署紧密关联的容器，确保它们在同一节点运行并协同工作。

4. Deployment和StatefulSet的区别？

Deployment用于无状态应用，Pod名称随机；StatefulSet用于有状态应用，提供稳定网络标识和有序部署/扩展/更新。

5. 如何实现Kubernetes集群的高可用？

多master节点，使用负载均衡；etcd集群（奇数节点）；关键组件多副本部署；使用PVC保证数据持久化。

面试高频问题

1. Prometheus的数据模型是什么样的？

时间序列数据，由指标名称(metric name)和标签(labels)唯一标识，每个时间序列是一组带有时间戳的值。

2. 如何设计一个完整的监控体系？

基础设施监控→应用监控→业务监控→用户体验监控，结合日志、指标、追踪数据，建立完善的告警机制和可视化。

3. 什么是PromQL？举例说明其用途。

Prometheus查询语言，用于从时间序列数据中查询和聚合数据。例如：http_requests_total{status="200"} 表示所有状态码为200的HTTP请求总数。

4. 如何处理监控系统的告警风暴？

设置告警抑制和分组，按业务优先级排序，实现告警聚合，避免重复告警，建立告警升级机制。

5. ELK Stack中各组件的作用是什么？

Elasticsearch：存储和索引日志；Logstash：收集和处理日志；Kibana：日志可视化和查询分析。

面试高频问题

1. MySQL主从复制的原理是什么？

主库开启binlog，从库通过IO线程读取主库binlog写入relay log，SQL线程执行relay log中的事件，实现数据同步。

2. 如何优化MySQL查询性能？

添加合适的索引，避免全表扫描；优化SQL语句，避免使用select *；分库分表；调整MySQL参数；增加缓存等。

3. Redis的持久化机制有哪些？区别是什么？

RDB：定时快照，性能好但可能丢失数据；AOF：记录所有写操作，数据安全性高但文件大，恢复慢。

4. 什么是数据库事务的ACID特性？

原子性(Atomicity)、一致性(Consistency)、隔离性(Isolation)、持久性(Durability)。

5. 如何处理MySQL主从延迟？

优化主库写入性能；升级从库硬件；使用并行复制；避免大事务；考虑使用中间件如MyCat、ProxySQL等。

面试高频问题

1. TCP三次握手和四次挥手的过程是什么？

三次握手：客户端发送SYN→服务器回SYN+ACK→客户端回ACK；四次挥手：主动方发FIN→被动方回ACK→被动方发FIN→主动方回ACK。

2. VLAN的作用是什么？如何实现跨VLAN通信？

作用：隔离广播域，增强网络安全性和管理效率。跨VLAN通信需通过路由器或三层交换机实现。

3. DNS的工作原理是什么？如何排查DNS故障？

原理：递归查询+迭代查询，从根域名服务器逐级查询到权威服务器。排查：使用nslookup/dig，检查/etc/resolv.conf，查看DNS缓存。

4. 什么是NAT？有哪些类型？

网络地址转换，将私有IP转换为公有IP。类型：静态NAT、动态NAT、PAT（端口地址转换）。

5. 如何使用tcpdump抓取HTTP请求的包？

tcpdump -i eth0 tcp port 80 -A （抓取80端口的TCP包并以ASCII码显示）

面试高频问题

1. IaaS、PaaS、SaaS的区别是什么？

IaaS提供基础设施（服务器、存储、网络）；PaaS提供开发平台（数据库、中间件）；SaaS提供成品软件（如Office 365）。

2. 什么是VPC？如何在VPC中实现不同子网间的通信？

VPC是隔离的虚拟网络环境。同一VPC内不同子网通过路由表默认互通；跨VPC需通过对等连接或VPN实现。

3. 云服务中的自动扩缩容原理是什么？如何配置？

基于预设指标（CPU使用率、请求数）自动增加/减少实例数量。配置：定义伸缩组、伸缩策略（触发条件）、最大/最小实例数。

4. 对象存储与块存储的区别？各自适用场景？

对象存储：按对象存储，适合非结构化数据（图片、日志），如S3/OSS；块存储：按块存储，适合数据库、虚拟机磁盘。

5. 如何设计云环境下的高可用架构？

多可用区部署，负载均衡分发流量，自动扩缩容应对负载变化，数据多副本存储，跨区域备份，故障自动转移。

面试高频问题

1. 如何防范SQL注入攻击？

使用参数化查询，避免拼接SQL语句；输入验证与过滤；最小权限原则配置数据库账户；使用WAF防护。

2. Linux系统有哪些常见的安全加固措施？

禁用root远程登录，使用sudo；设置强密码策略；开启防火墙和SELinux；定期更新系统；删除无用账户和服务。

3. 什么是DDoS攻击？如何防御？

分布式拒绝服务攻击，通过大量请求耗尽目标资源。防御：流量清洗、CDN分流、高防IP、速率限制、黑洞路由。

4. 如何进行日志审计与安全事件分析？

集中收集系统、应用、网络日志；使用ELK等工具分析；设置异常行为告警（如多次登录失败、敏感文件访问）；定期审计。

5. 什么是零信任架构？其核心原则是什么？

默认不相信任何访问者（内部/外部），需持续验证。核心原则：最小权限、持续验证、深度防御、假设已被入侵。

面试高频问题

1. Ansible的工作原理是什么？与SaltStack有何区别？

Ansible基于SSH协议无代理工作，使用YAML编写Playbook；SaltStack需安装minion代理，支持更实时的通信，性能更好。

2. 什么是基础设施即代码(IaC)？有什么优势？

将基础设施配置以代码形式管理。优势：版本控制、环境一致性、可重复部署、自动化执行、便于协作。

3. 如何设计一个CI/CD流水线？

代码提交→自动构建→自动化测试（单元/集成）→代码质量检查→构建镜像→部署到测试环境→人工确认→部署到生产环境。

4. 用Ansible写一个Playbook，实现Nginx的安装与启动

- name: Install and start Nginx
  hosts: webservers
  tasks:
   - name: Install nginx
    package: name=nginx state=present
   - name: Start nginx service
    service: name=nginx state=started enabled=yes

5. 自动化运维可能面临的挑战有哪些？

初期学习成本高；复杂场景的脚本编写难度大；环境差异导致的兼容性问题；自动化故障的影响范围扩大。

iSCSI: 块存储网络协议
NFS/SMB: 文件共享协议
Ceph: 分布式统一存储
rsync: 数据同步工具

RAID级别对比

RAID0: 条带化，高性能无冗余
RAID1: 镜像，100%冗余，读性能好
RAID5: 分布式校验，允许单盘故障
RAID6: 双校验，允许双盘故障
RAID10: 镜像+条带，高性能高冗余

面试高频问题

1. 块存储、文件存储、对象存储的主要区别是什么？

块存储：按块访问，适合数据库；文件存储：按文件/目录访问，适合共享文件；对象存储：按唯一ID访问，适合海量非结构化数据。

2. 不同RAID级别的适用场景？

RAID0用于临时数据；RAID1用于系统盘；RAID5用于一般业务数据；RAID6用于大容量存储；RAID10用于高性能高可靠场景（如数据库）。

3. LVM的主要功能是什么？如何创建一个LVM逻辑卷？

功能：动态调整磁盘容量，灵活管理。步骤：创建物理卷(pvcreate)→卷组(vgcreate)→逻辑卷(lvcreate)→格式化挂载。

4. Ceph存储的核心组件有哪些？

Monitor（集群监控）、OSD（对象存储守护进程）、MDS（元数据服务器，用于CephFS）、RGW（对象网关，兼容S3/Swift）。

5. 如何提高存储系统的性能？

选择高性能存储介质（SSD/NVMe）；合理规划RAID级别；优化文件系统参数；增加缓存；使用存储虚拟化和分层存储。

架构设计

核心知识点

  • 架构模式: 单体架构、微服务、Serverless、微内核、分层架构的特点
  • 高可用设计: 冗余设计、故障转移、负载均衡、限流降级、熔断机制
  • 可扩展性设计: 水平扩展vs垂直扩展，无状态服务设计，数据分片与分区
  • 微服务架构: 服务注册与发现，API网关，服务通信（同步/异步），服务治理
  • 数据架构: 分库分表，读写分离，数据一致性，缓存策略，数据备份
  • 灾备设计: RPO/RTO指标，多活架构，跨区域备份，灾难恢复流程

关键组件与模式

负载均衡

LVS, Nginx, F5；策略：轮询、加权、IP哈希、最少连接

缓存系统

Redis, Memcached；策略：LRU, LFU；缓存穿透/击穿/雪崩

消息队列

Kafka, RabbitMQ, RocketMQ；解耦、异步、削峰填谷

面试高频问题

1. 微服务与单体架构的优缺点对比？

微服务：独立部署、技术栈灵活、可单独扩展，但复杂度高、分布式问题多；单体：简单易维护，但扩展受限、迭代慢。

2. 如何设计一个高可用的系统架构？

多副本部署，避免单点；负载均衡分发流量；服务熔断与降级；数据多副本存储；自动故障检测与转移；完善监控告警。

3. 缓存常见的问题（穿透、击穿、雪崩）如何解决？

穿透：布隆过滤器；击穿：互斥锁、热点数据永不过期；雪崩：过期时间加随机值、服务熔断降级、多级缓存。

4. 什么是CAP定理？分布式系统如何取舍？

一致性(C)、可用性(A)、分区容错性(P)三者不可兼得。分布式系统必须保证P，通常在CP（如ZooKeeper）或AP（如Eureka）之间取舍。

5. 如何设计一个支持高并发的电商秒杀系统？

前端限流；CDN静态资源；队列削峰；库存预扣减；缓存热点商品；服务集群扩展；数据库读写分离与分库分表。

故障排查

核心知识点

  • 排查方法论: 分层排查法（应用→系统→网络→硬件），对比法，替换法，排除法
  • 系统故障: CPU/内存/磁盘/网络瓶颈识别，进程异常，服务启动失败
  • 网络故障: 连通性问题，端口不通，DNS解析异常，带宽耗尽，延迟过高
  • 应用故障: 日志分析，堆栈跟踪，依赖服务异常，配置错误，内存泄漏
  • 数据库故障: 连接数满，锁等待，慢查询，主从同步异常，磁盘空间不足
  • 应急响应: 故障定级，快速止损，根因分析，恢复流程，事后复盘

常用排查工具与命令

系统与性能排查

top/htop: 进程与资源监控
vmstat/iostat: 系统性能统计
free/df/du: 内存与磁盘使用
strace: 系统调用跟踪
lsof: 打开文件列表

网络与应用排查

ping/traceroute/mtr: 网络连通性
tcpdump/wireshark: 流量抓包
netstat/ss: 网络连接状态
curl/telnet/nc: 服务端口测试
journalctl/systemctl: 服务日志

面试高频问题

1. 服务器CPU使用率过高，如何排查原因？

top命令查看占用CPU高的进程；使用ps查看进程详情；strace跟踪进程系统调用；检查是否有死循环或异常线程。

2. 网站访问慢，可能的原因有哪些？如何排查？

原因：网络延迟、服务器负载高、数据库慢查询、资源加载多。排查：浏览器开发者工具分析加载时间；监控服务器资源；检查慢查询日志；网络链路测试。

3. 数据库连接数满了，如何处理？

临时增加max_connections；排查连接泄露（未释放的连接）；优化连接池配置；分析慢查询导致的连接阻塞；考虑读写分离分担压力。

4. 两台服务器之间无法通信，如何排查？

检查网络连通性（ping）；确认端口是否开放（telnet/nc）；查看防火墙规则（iptables）；检查路由配置（route）；抓包分析（tcpdump）。

5. 什么是事后复盘（Postmortem）？包含哪些内容？

故障后的结构化分析过程。内容：故障时间线、影响范围、根本原因、解决方案、预防措施、经验教训，避免重复发生。
